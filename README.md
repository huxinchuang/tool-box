# 货车运输记账工具 - 微信小程序

## 项目简介

本工具专为货车运输记账设计，支持逐条录入运输记录，自动计算总价，智能合并相同车次的数据，并生成可保存的 Excel 风格表格图片。适用于需要快速记录运费、核算欠款的场景。

---

## 主要功能

### 1. 记录录入
- **输入字段**：日期、车号、规格、件数、重量（吨）、单价（元/吨）
- **自动计算**：总价 = 重量 × 单价
- **智能补全**：
  - 若**日期为空**，自动继承上一条记录的日期（首条记录允许为空，但会留空）
  - 若**车号为空**，自动继承上一条记录的车号（首条车号**必须填写**）
- **提交后清空**：录入一条记录后，所有输入框清空，便于连续录入

### 2. 数据合并规则（重点）
当**连续两条或以上记录**满足以下**四个字段完全相同**时：
- 日期
- 车号
- 重量
- 单价

则：
- **后续记录**的“重量/吨”、“单价”、“总价”单元格显示为空（不显示数值）
- 但后续记录的“规格”、“件数”正常显示
- 在**合计金额**和**运费重量**计算中，只计**该组的第一条记录**（重复的后续记录不重复累加）

> 示例：  
> 记录 A：2026-03-12，京A123，水泥，10吨，50元/吨，总价500  
> 记录 B：2026-03-12，京A123，水泥（另一规格），10吨，50元/吨，总价500  
> 则表格中 B 的重量、单价、总价显示为空，合计金额只加一次500元。

### 3. 运费计算
- **运费重量**：遍历所有合并组，每组只取一次重量（去重求和）
- **运费单价**：用户可自定义（默认30元/吨）
- **运费总价** = 运费重量 × 运费单价
- 运费行显示在表格末尾，行背景为浅黄色

### 4. 底部结欠行
- 显示格式：`{最新日期}止合计结欠货款 {总金额} 元`
- **最新日期**：取所有非空日期中的最大值（若无日期则显示“至今”）
- **总金额** = 所有合并组的总价之和 + 运费总价

### 5. 附录功能
- 提供多行文本输入框，可填写备注信息（如“以上数据已核对”）
- 内容**自动保存**到本地缓存，下次打开小程序自动加载
- 附录文本会显示在表格图片的最底部

### 6. 生成图片
- 点击“生成表格图片”按钮，将当前预览表格（包含所有数据行、运费行、结欠行、附录）渲染为 PNG 图片
- 图片尺寸自动适应内容，最大不超过 4096×4096
- 生成后显示缩略图，支持**点击预览**和**保存到相册**

### 7. 删除记录
- 点击“删除上一行”按钮，删除最后一条**原始录入记录**（非合并后的行）
- 删除后自动重新计算合并与合计

## 边界情况与规则定义

| 场景 | 处理方式 |
|------|----------|
| **首条记录车号为空** | 提示“请填写车号（首条记录不能为空）”，拒绝添加 |
| **日期格式错误** | 支持 `20260312` 或 `2026-03-12`，其他格式提示错误 |
| **日期为空** | 存储为空字符串，预览时显示空白；空日期之间可以相互合并（视为相同） |
| **车号为空（非首条）** | 自动继承上一条记录的车号 |
| **重量或单价不是数字** | 提示“重量和单价必须是数字”，拒绝添加 |
| **规格、重量、单价任一为空** | 提示“规格、重量、单价不能为空”，拒绝添加 |
| **件数为空** | 存储为空字符串，正常显示 |
| **合并条件** | 仅当**连续且完全相同**（日期、车号、重量、单价）才触发隐藏，不跨间隔 |
| **合计金额计算** | 基于合并组（每组只计一次），而非原始记录条数 |
| **运费重量计算** | 同样基于合并组去重（相邻且重量+单价相同只计一次） |
| **最新日期获取** | 取所有非空日期的最大值（按字符串排序即可，格式 YYYY-MM-DD） |
| **图片尺寸限制** | 表格总宽或总高超过 4096px 时提示“表格过大”，拒绝生成 |
| **生成图片失败** | 显示错误信息（如 canvas 获取失败、路径无效等） |
| **保存图片权限** | 若用户拒绝授权，弹窗引导前往设置开启 |

## 技术要点

- 基于微信小程序原生框架（基础库 ≥ 2.9.0，支持 Canvas 2D）
- 数据存储：`wx.setStorageSync` / `wx.getStorageSync` 保存运费单价和附录
- 图片生成：`canvas` 绘制表格，`wx.canvasToTempFilePath` 导出临时文件
- 日期处理：统一标准化为 `YYYY-MM-DD`，显示转为 `YYYY年M月D日`

## 如何运行

1. 下载本仓库代码
2. 使用微信开发者工具打开项目
3. 修改 `app.json` 中的 `pages` 顺序，确保 `"pages/index/index"` 为首页
4. 编译运行即可

---

## 注意事项

- 本工具依赖 Canvas 2D，请确保微信客户端版本支持
- 生成的图片仅保存在本地临时目录，需用户主动“保存到相册”才能持久化
- 运费单价和附录的本地缓存与设备绑定，卸载小程序后可能清除
